# Use Cases & Examples (Draft)

Below are example scenarios across industries and stages.

## A. Fintech — Early Concept
**Feature**: “One-tap credit line increase in mobile app.”
**Personas**: PM, DPO, DS, New User
**Insights**:
- PM: How will we prevent accidental taps? Add confirm + undo.
- DPO: Clarify consent, data retention, regulatory disclosures.
- DS: Define success metric (approved increases with <X% delinquency).
- New User: Wants plain-language tooltips and a demo video.

## B. Healthtech — Detailed Design
**Feature**: “Symptom checker with photo upload.”
- A11y: Alternative text for results; voiceover-friendly steps.
- DPO: Secure storage, strict retention, explicit medical disclaimers.
- Power User: Batch uploads + keyboard shortcuts.

## C. SaaS — Pre-Launch Validation
**Feature**: “Bulk CSV import with field mapping.”
- PM: Guardrails for partial failures; segment-level KPIs post-import.
- DS: Track error distributions to improve mapping models.
- Power User: Save mapping templates; CLI parity.

## D. Content Platform — Workflow
**Feature**: “Scheduled posts with multi-account cross-posting.”
- PM: Rollout plan; SLAs; scheduling reliability.
- A11y: Focus order and keyboard navigability in the scheduler.
- New User: Sample templates and preview before posting.
